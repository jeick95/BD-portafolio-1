Control de Versiones

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear cambios en el código fuente durante el desarrollo de software.

Ver Manual de GitHub Ver Proyectos
# Configura tu identidad
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
# Inicia y confirma
git init
git add .
git commit -m "Primer commit"

Características Principales

Descubre por qué Git es esencial para cualquier desarrollador

Control de Versiones

Mantén un historial completo de todos los cambios en tu código. Vuelve a cualquier versión anterior con facilidad.

Ramificación (Branching)

Crea ramas para trabajar en nuevas funcionalidades sin afectar el código principal. Fusiona cuando estés listo.

Colaboración

Trabaja en equipo de manera eficiente. Cada desarrollador puede tener su propia copia del repositorio completo.

Revertir Cambios

Comete errores? No hay problema. Git te permite revertir cambios de forma segura sin perder el historial.

Rendimiento

Git está optimizado para ser rápido. Las operaciones locales son instantáneas y las remotas están optimizadas.

Seguridad

El contenido de los archivos y las relaciones entre commits, ramas y etiquetas están protegidos con SHA-1.

Comandos Básicos

Aprende los comandos esenciales de Git

1

Inicializar un Repositorio

El primer paso es inicializar Git en tu carpeta de proyecto para comenzar a rastrear cambios.

git init
2

Agregar Archivos

Añade archivos al área de preparación (staging) antes de confirmarlos.

git add archivo.txt    git add .
3

Confirmar Cambios

Crea una confirmación con un mensaje descriptivo de los cambios realizados.

git commit -m "Descripción del cambio"
4

Ver el Estado

Consulta qué archivos han sido modificados, eliminados o están sin seguimiento.

git status
5

Ver el Historial

Muestra todos los commits realizados con sus detalles y cambios.

git log    git log --oneline
